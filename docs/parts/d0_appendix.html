付録　ビルド設定・拡張のヒント・早見表

A.1　ビルド設定

開発環境は Visual Studio 2022 / C++20 / DirectX 11、構成は Debug / Release の x64 です。ゲーム層を組む上で、特に注意すべき プロジェクト設定が1つあります。

/utf-8 コンパイルフラグ 本プロジェクトのソースは、日本語コメントを含むBOMなしUTF-8で書かれています。 MSVCはBOMが無いソースを既定でShift-JISと解釈するため、そのままでは日本語コメントや 文字列リテラルが文字化け・コンパイルエラーになります。プロジェクトの追加オプションに /utf-8 を指定することで、ソースをUTF-8として正しく扱えます。これは ゲーム層を書き始める前（構築ステップ1）に必ず設定してください。

ビルドはVisual Studio上のほか、MSBuildでも行えます。新しいソースファイルを追加した ときは、.vcxproj に ClCompile / ClInclude エントリを、 .vcxproj.filters にフィルタ（ソリューションエクスプローラ上の仮想フォルダ） エントリを追加します。

A.2　拡張のヒント

本設計の狙いは「拡張のしやすさ」です。代表的な3つの拡張を、レシピとして示します。

新しいアビリティを追加する

  1. どのフックでルールを書き換えるかを決める（例：思考時間なら ITurnTimeProvider）。
  2. game/ability/abilities/ に ○○.h/.cpp を作り、 Ability と対象フックを多重継承する。
  3. コンストラクタで名前・説明・レアリティを設定する。
  4. パッシブなら RegisterTo でレジストリへ登録（単一フックは Set○○ でチェイン、複数フックは Add○○）。 アクティブなら activatable = true とし Activate 等を実装する。
  5. フックのメソッドを実装する。チェイン型なら「自分の関心事だけ変え、残りは chained へ委譲」を守る。
  6. ability_pool.cpp の CreateAll() に追加して、報酬抽選の 候補に含める。
  7. .vcxproj / .vcxproj.filters にファイルを登録する。

既存のパラメータ違いで済むなら、「余裕」が「熟考」を継承したように、既存 アビリティを継承してコンストラクタだけ書き換えるのが最も簡単です。

新しいボスを追加する

  1. ボスのギミックを、アビリティとして実装する（Ability＋必要なフック）。
  2. Boss を継承した派生クラスを作り、GetBossAbilities() で そのギミックを、GetRewardAbility() で撃破報酬（ギミックの弱化版）を返す。
  3. ゲームシーンの LoadBoss() が、ボス番号に応じて生成するボスを 切り替えられるようにする（プロト版は固定生成）。

新しいフック（ルールの種類）を追加する

盤面サイズフック IBoardModifier を後から追加したときの手順が、そのまま 雛形になります。

  1. hooks.h にインターフェイスを定義する。
  2. default_hooks に標準実装を追加し、AbilityRegistry::ResetToDefaults でセットする。
  3. AbilityRegistry に保持メンバと Set/Add・呼び出しヘルパを足す。
  4. ゲーム本体の該当箇所を、レジストリ経由の呼び出しへ置き換える。

A.4　処理フロー早見表

本書で解説した主要なフローを、最後にまとめます。

[アプリ起動] WinMain → 基盤層初期化 → Scene_Initialize → メインループ [1フレーム] Scene_Update → Scene_Draw → Direct3D_Present → Scene_Refresh [試合開始] ResetMatch → LoadBoss（アビリティ登録）→ GetBoardSize/Resize → BeginTurn [1回の着手] 入力 or AI → PlaceAndEvaluate → OnPlace（設置フック発火・animPhasesへ移動を積む） → StartEffectAnimation（フェーズ展開） → EvaluateAfterPlace（両陣営の勝敗→引分→追加着手→ターン終了） [ボス撃破後] 結果画面で「報酬を受け取る」→ IncrementBoss → GenerateRewardChoices → Scene::REWARD（3択）→ 選択 → PlayerAbilitiesへ追加 → Scene::GAME
おわりに 本書は、1つの〇×ローグライトを題材に「ルールを差し替え可能にする設計」を解説して きました。フックという1つの考え方が、アビリティ・ボスギミック・盤面構造・UIまでを 貫いていることを読み取っていただけたなら幸いです。同じ設計は、カードゲームや 他のローグライトにも応用できます。本書が、拡張に強いゲームを設計するための 一例として役立つことを願っています。
